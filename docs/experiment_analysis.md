It separates deterministic no-key ablations from the API-backed benchmark
pipeline so the project can be graded even when cloud keys are unavailable.

## Member 1: Search Improvement

Command:

```bash
.venv/bin/python tools/member1_search_ablation.py --write docs/member1_search_ablation.md
```

Result summary:

| strategy | accuracy | avg_nodes | avg_simulations | avg_model_calls | avg_pruned |
|---|---:|---:|---:|---:|---:|
| baseline | 0/3 | 3.0 | 3.0 | 6.0 | 0.0 |
| adaptive | 3/3 | 1.0 | 1.0 | 2.0 | 2.0 |

Interpretation: the adaptive strategy avoids the distracting first candidate,
selects the promising final step, and stops earlier. This supports the Member 1
claim that search-side pruning and early stopping improve both quality and
efficiency in controlled cases.

## Member 2: Scoring Improvement

Command:

```bash
.venv/bin/python tools/member2_scoring_ablation.py --write docs/member2_scoring_ablation.md
```

Result summary:

| metric | value |
|---|---:|
| Mis-rankings corrected | 3/3 |
| Old scorer top choice | vague guess in every case |
| Hybrid scorer top choice | mathematically useful step in every case |

Interpretation: the hybrid scorer combines a PPM-compatible learned score with a
deterministic math verifier. When the learned score is noisy, the verifier
penalizes guessing and rewards algebraic/calculus progress. This satisfies the
scoring-side innovation and provides error-analysis examples.

## End-to-End Comparison Demo

Run:

```bash
.venv/bin/streamlit run app.py
```

The UI runs the same problem through:

1. Baseline MCTS.
2. Improved adaptive MCTS with verifier scoring.
3. Improved adaptive MCTS with hybrid PPM+verifier scoring, if a checkpoint is supplied.

Displayed evidence includes selected first step, reasoning trajectory, model
calls, pruned branches, and latency. The same functionality is exposed through
FastAPI:

```bash
.venv/bin/uvicorn backend.main:app --reload
```

## Full Benchmark Protocol

The full benchmark requires cloud API access and, for dataset loading,
`datasets` and `sentence-transformers`.

```bash
.venv/bin/python experiments/run_experiment.py \
  --train 20 \
  --test 10 \
  --simulations 3 \
  --output data/experiment_results.json
```

This pipeline:

1. Loads MATH Level 5 problems.
2. Collects adaptive-MCTS trajectories.
3. Builds preference pairs.
4. Trains a local PPM.
5. Compares baseline MCTS, adaptive MCTS, and adaptive MCTS plus PPM top-k pruning.

For MATH/OlympiadBench strategy comparisons:

```bash
.venv/bin/python eval.py --dataset math --n 20 --strategy direct --model openai
.venv/bin/python eval.py --dataset math --n 20 --strategy mcts --model openai
.venv/bin/python eval.py --dataset math --n 20 --strategy mcts+ppm --model openai --ppm-checkpoint checkpoints/ppm.pt
```

## Limitations

- The deterministic ablations prove mechanism and integration, but they are not
  a replacement for large-scale external benchmark runs.
- A trained PPM checkpoint is generated by the provided scripts and is not
  committed by default because it can be regenerated from trajectory data.
- API-backed accuracy depends on model choice, API availability, and random
  sampling from the benchmark.
